Reports config keys missing the + prefix in dbt_project.yml.

Under node-type blocks (models, seeds, snapshots, tests, analyses), configuration values must use the + prefix to distinguish them from directory path groupings.

For example, instead of:


models:
  my_project:
    materialized: table

Use:


models:
  my_project:
    +materialized: table

See MissingPlusPrefixDeprecation for details.
