Reports resource properties that should be nested under config: in dbt YAML files.

Since dbt v1.10, the following properties must be placed under config: on resource definitions:

  • freshness (on sources)
  • meta (on any resource)
  • tags (on any resource)
  • docs (on any resource)
  • group (on models, seeds, snapshots)
  • access (on models)

For example, instead of:


sources:
  - name: ecom
    freshness:
      warn_after:
        count: 24
        period: hour

Use:


sources:
  - name: ecom
    config:
      freshness:
        warn_after:
          count: 24
          period: hour

See PropertyMovedToConfigDeprecation for details.
